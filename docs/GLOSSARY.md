# Glossary (Accessible Terms)

This glossary explains project terms in simple language.

## A

API
- A way for one software system to talk to another system.

Alert
- A structured intelligence item generated by the pipeline to highlight a potentially important event.

Analyzer Stage
- Pipeline stage that classifies each signal (category, sentiment, location, keywords).

Audit Log
- Historical record of actions taken in the system (for transparency and compliance).

## B

Backend
- Server-side application that collects, processes, and returns intelligence data.

## C

Category
- Event type assigned to an alert (for example PROTEST, VIOLENCE, ACCIDENT).

Collector Stage
- Pipeline stage that fetches raw public signals from sources like APIs, RSS, search, and scraping.

Confidence Score
- Numeric estimate (0-100) of how trustworthy a signal appears, based on source and content cues.

CORS
- Browser security setting controlling which frontend origins are allowed to call backend APIs.

## D

De-duplication (Dedupe)
- Removing repeated versions of the same signal so analysts do not see duplicates.

## E

Evidence
- Source excerpts and links attached to an alert showing why the alert was generated.

Escalation Probability
- Estimated chance (0-100) that a situation may become more serious soon.

## F

Fallback Mode
- Backup report generation mode used when Gemini is unavailable.

Frontend
- User interface application where analysts view alerts and run workflows.

## G

Gemini
- Google AI model used here to generate operational intelligence briefings.

## H

Health Endpoint
- API endpoint (`/health`) that reports whether backend and keys are available.

## I

Impact
- Estimated severity of outcomes if an alert situation continues or worsens.

## L

LLM (Large Language Model)
- AI model trained on large text data; used in this project for briefing/report generation.

## M

Meta
- Extra technical information in API results (for example report mode, model, source counts).

## O

Orchestrator
- Component that runs pipeline stages in order and assembles final result payload.

OSINT
- Open Source Intelligence: analysis of publicly available information.

## P

Pipeline
- Ordered set of processing stages from data collection to final report generation.

Predictor Stage
- Stage that computes risk level, confidence, impact, escalation probability, and actions.

## R

Realtime Stream (SSE)
- Continuous server-to-browser event stream used to show pipeline progress live.

Reporter Stage
- Stage that generates final report text (Gemini or fallback).

Risk Level
- Priority category showing severity: LOW, MEDIUM, HIGH.

## S

SSE (Server-Sent Events)
- Web mechanism where server pushes event updates to client over a long-lived HTTP connection.

Sentiment
- Tone inferred from text (PANIC, AGGRESSION, NEUTRAL, TENSE).

Signal
- A single collected raw information item (news article, RSS entry, search result, scraped item).

Source Validity
- Label derived from confidence (VERIFIED, MIXED, UNVERIFIED).

## T

Topic
- Search phrase entered by user to drive collection and analysis.

## U

UI Mapping
- Conversion from backend field names to frontend field names for display.

## W

Web Scraping
- Extracting text and links from publicly available web pages.

Why Triggered
- Human-readable explanation in each alert that describes why scoring/rules marked it as important.
